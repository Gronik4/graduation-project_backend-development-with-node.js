  API Модуля «Бронирование»:
    POST /api/client/reservations - {Доступно только аутентифицированным пользователям с ролью client}
      {401 - если пользователь не аутентифицирован; 403 - если роль пользователя не client; 400 - если номера с указанным ID не существует или он отключён.}

    GET /api/client/reservations - {Доступно только аутентифицированным пользователям с ролью client}
      {401 - если пользователь не аутентифицирован; 403 - если роль пользователя не client;}

    DELETE /api/client/reservations/:id - {Доступно только аутентифицированным пользователям с ролью client.}
      {
        401 - если пользователь не аутентифицирован; 403 - если роль пользователя не client; 403 - если ID текущего пользователя не совпадает с ID пользователя в брони;
        400 - если брони с указанным ID не существует.
      }

    GET /api/manager/reservations/:userId - {Доступно только аутентифицированным пользователям с ролью manager.}
      {401 - если пользователь не аутентифицирован; 403 - если роль пользователя не manager;}

    DELETE /api/manager/reservations/:id - {Доступно только аутентифицированным пользователям с ролью manager.}
      {401 - если пользователь не аутентифицирован; 403 - если роль пользователя не manager; 400 - если номера с указанным ID не существует}

  API Модуля «Аутентификация и авторизация»:
    POST /api/auth/login - {Доступно только аутентифицированным пользователям.}
      {401 - если пользователя с указанным email не существует или пароль неверный.}

    POST /api/auth/logout - {Доступно только аутентифицированным пользователям.}
      
    POST /api/client/register - {Доступно только не аутентифицированным пользователям.}
      {400 - если email уже занят.}
