## "Дипломный проект по специальности «Backend-разработка на Node.js»"  
Дипломная работа за весь курс "Node.js для backend-разработки".  
#### Автор: Гречишников О.Н. группа NDJS-ASINHR   
### 'Описание'  
  Дипломный проект представляет собой сайт-агрегатор просмотра и бронирования гостиниц. Задача заключается в разработке бэкенда для сайта-агрегатора с реализацией возможности бронирования гостиниц на диапазон дат.  
  Полный текст задания к дипломному проекту: [здесь](https://github.com/netology-code/ndjs-diplom)   
### Цели проекта:  
  1. Разработка публичного API.
  2. Разработка API пользователя.
  3. Разработка API администратора.
  4. Разработка чата консультанта.  
#### При разработке проекта использовался технологический стек:  
  - Node.js;
  - Nest.js;
  - MongoDB;
  - WebSocket;
  - Event Emitter
  - RxJS;
  - Passport.js;
  - Express.js;
  - Multer;
  - bcrypt.js;
  - Moment.js;
  - Nano ID.  
  - Docker

#### Рекомендации по запуску проекта:  
  
Для успешного запуска проекта необходимо выполнить следующее:  
#### Вариант №1 (Упрощенный).  
  Если не стоит цели создавать новую базу данных, а воспользоваться имеющейся, нужно сделать следующее:  
  Командой: **Bash/командная строка: `docker pull gronik4/diploma-project_on_nodejs:v1.2.15`** - такой командой скачается на Вашу машину готовый Docker образ.  
  Командой:  **Bash/командная строка:  `docker run -p 3001:3001 -it --rm  gronik4/diploma-project_on_nodejs:v1.2.15`** - запустить готовый проект с уже подключенной MongoDB.  
  **Однако!!** Следует иметь в виду, что:  
  1. В этом случае, в базе данных уже предустановлены некотрые учетные записи, конкретно какие - смотри файл Initial-data.txt  
  2. При создании учетной записи администратором, пароль, указанный в Body - параметрах, подменяется сгенерированным случайно!!! И ни где не указывается!!!(Основание - требование задание на проектирование.) Соответственно, узанть его не было бы ни какой возможности, если бы автор проекта не вывел бы его в соответствующем месте в консоль!
  3. В каждом модуле базы данных есть некоторые отладочные записи.  
  4. Все обращения, согласно указанным в задании на разработку проекта защищены соответствующими Guard - файлами.  
    
#### Вариант №2 (Подключение своей базы данных).  
  В таком случае, нужно сделать следующее:  
  1. Командой:  **Bash/командная строка: `git clone https://github.com/Gronik4/graduation-project_backend-development-with-node.js.git`** - скачать себе на машину готовый проект.  
  2. На основе файла .env-example создать свой .env - файл, заполнив его соответствующими данными. (Затем файл .env-example - можно удалить).  
  3. В файле users.controller.ts в методе:
  ```JavaScript:
    @Post('/admin/users/')
    @UseGuards(AuthUserGuard)
    create(@Body() body: createUserDto): Promise<Partial<UserDocument> | null> {
      return this.userSRV.create(body);
    }
  ```  
  закомментировать строку: ``` @UseGuards(AuthUserGuard)```.  
  4. Запустить проект командой:**Bash/командная строка: `npm start | npm start:dev`**.  
  5. Создать учетную запись с ролью: 'admin'.  
  6. При создании учетной записи, обязательно учесть п2 варианта №1!!!! Пароль необходимо где-то сохранить!! 
  7. Убедившись, что все отработало корректно, остановить проект: ```ctrl-c```.  
  8. Раскомментировать строку: ``` @UseGuards(AuthUserGuard)``` и не забыть сохранить изменения.  
  При необходимости создания Docker образа:  
  9. Командой:**Bash/командная строка: `docker compose build`** - создать образ.  
  10. Командой:**Bash/командная строка: `docker compose up -d`** - из образа создать и запустить контейнер, для проверки работоспособности.
